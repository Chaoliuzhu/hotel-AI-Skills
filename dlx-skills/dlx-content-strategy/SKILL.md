## 选题评估框架（content-strategy + 德胧融合）

### 热点价值评估

```
1. 相关性：这个热点与德胧酒店有多大关系？
   → 高：可直接结合酒店产品
   → 中：需要找到切入角度
   → 低：跳过

2. 时效性：这个热点的窗口期有多长？
   → 突发热点（2小时内响应）
   → 常规热点（提前1天规划）
   → 长期热点（纳入内容日历）

3. 受众匹配：德胧的目标客群是否关注这个热点？
   → 商旅客人 vs 度假客人 vs 本地散客
```

### 内容形式选择

| 热点类型 | 推荐形式 |
|----------|----------|
| 季节美景 | 场景种草+酒店套餐 |
| 节假日活动 | 活动联动+限时优惠 |
| 突发热点 | 快讯+情感共鸣 |
| 美食相关 | 攻略+餐厅推荐 |
| 亲子相关 | 攻略+亲子房套餐 |

## 执行流程

### Step 1：获取热点

搜索"天津滨海新区 + [当前时间] 热点"：
- 节假日文旅活动
- 本地新闻/话题
- 社交媒体热门

### Step 2：评估过滤

用上方的评估框架判断是否适合做内容。

### Step 3：绑定德胧产品

找到热点与酒店产品的结合点：
- 地理位置（离热点近）
- 产品匹配（亲子/美食/商务）
- 体验差异化（服务/环境/设施）

### Step 4：生成内容日历

输出格式：
```
📅 [日期] 内容日历

周一：xxx
周二：xxx
...

本周重点选题：
1. [热点A] → [内容方向] → [平台]
2. [热点B] → [内容方向] → [平台]
```

## 容错机制

### 异常处理

- **web_search 搜索失败（网络问题或无结果）**：使用内置的全年时令热点日历作为默认内容来源；标注「⚠️ 实时热点未能获取，使用时令热点替代」
- **用户指定的日期/节假日不在时令日历中**：映射到最近的时令节点；例：10月某普通周末→使用"秋季海鲜季"主题
- **dlx-copywriting 调用失败**：降级为"内容框架"输出（选题+平台+形式），不生成完整文案；标注「文案生成已降级，建议手动补充」
- **内容日历输出超过Token限制**：自动截断至7日内容，剩余日期折叠为「...及后续常规内容」；询问用户是否需要展开
- **热点与酒店定位不匹配（如追夜店热点）**：自动过滤并标注「已跳过不匹配热点：[xxx]，理由：与商务+度假双定位不符」

### 决策边界

| 情况 | 决策 |
|------|------|
| 实时热点获取失败 | 使用内置时令日历，保证内容产出不中断 |
| 用户未指定时间范围 | 默认输出"本周+下周"内容日历（约14天） |
| 热点窗口期已过 | 跳过已过期热点，自动补入下一个时令节点 |
| 平台与内容形式不匹配 | 自动调整；例：抖音热点不适合图文→改为小红书笔记 |
| 内容量过少（<3条/周） | 自动补充常规内容填充至7条（每日一条） |

### 检查点机制

每完成一个平台的评估输出状态快照：`📍 [平台] 评估完成 | 可用热点：X个 | 建议选题：X个 | 内容形式：XXX`
