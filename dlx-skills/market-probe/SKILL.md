# SKILL.md - 市场探针(Market Probe)

## 基本信息

- **技能名称**: market-probe
- **中文名**: 市场探针
- **版本**: v2.4
- **定位**: 酒店前端精准拓客情报系统。基于地理探针+工商探针+关系链探针,**以存量客户关系链为核心,优先实现"抄近道"式盘活**,同时探测新兴行业和上下游关联企业。
- **适用酒店**: 德胧旗下所有酒店
- **维护者**: 德胧AI龙虾军团
- **核心理念**: "选客"思维 - **有关系链的B级客户 > 无关联的A级客户**
- **整合来源**: dlx-customer-research(四大客群体系)+ scout-growth(RICE评分)+ 五刷方法论

## 选客四步法(整合dlx-customer-research)

**不是所有客户都要开发,有限精力优先投入最高价值的增量线索!**

| 步骤 | 内容 | 市场探针能力 |
|------|------|------------|
| **识** | 识别谁是目标客户(行业/规模/关系链) | 探针B规模探测 |

## "五刷"拓客方法论(销售一线实战)

**⚠️ 必须写入SKILL底层!这是选客的核心方法论!**

### 五刷方法

#### 1. 刷高德(地图探针)
**操作**:高德地图搜索企业关键字 → 带出关联分子公司
**工具**:百度地图Place API + 企查查公司链
**价值**:快速获取企业分支机构网络

#### 2. 刷微信首字母
**操作**:按微信好友首字母A-Z排序 → 挖掘被遗忘的客户
**工具**:微信(人工操作)+ AI辅助分析
**价值**:激活沉默客户,触发互动

#### 3. 刷上下游
**操作**:从一家核心企业 → 探测其供货商+销售渠道+客户
**工具**:企查查公司链(股东/分支机构/对外投资)
**价值**:央国企/大公司的上下游本身就是巨大的增量池

#### 4. 刷地图(片区扫描)
**操作**:按功能片区/园区 → 一个片区一个片区扫描
**工具**:百度地图POI区域检索
**价值**:集中火力,精准爆破

#### 5. 刷行业
**操作**:按行业分类 + 新质生产力分类 → 系统性扫描
**工具**:企查查行业搜索 + 百度地图关键词
**价值**:不遗漏任何行业赛道

### 标签交叉理论(核心洞察)

```
当一家企业被打上的标签越多 → 越应该被优先开发
```

| 标签类型 | 数据来源 |
|----------|---------|
| 消费频次高 | 开票数据 |
| 关系链近 | 存量客户的公司链 |
| 规模大 | 企查查参保人数 |
| 行业匹配 | 百度地图行业标签 |
| 地理近 | 地图POI距离 |
| 上下游关联 | 企查查公司链 |

**多标签交叉 = 高置信度开发目标**

## 四大探针模块

### 🔍 探针0:存量客户获取(必须首先执行)

**目标**: 获取德胧该酒店的存量客户名单

**执行方式**:
1. 从德胧PMS系统导出
2. 或从企业卡动销表格筛选该酒店客户
3. 或询问用户直接提供

**输出**: 存量客户名称列表

---

### 🔍 探针C:公司链探测(关系链探针)

**目标**: 对存量客户探测其**关联公司链**,发现增量机会

**核心逻辑**:
```
存量客户
├── 股东信息(向上穿透)→ 母公司/实际控制人
├── 对外投资(向下穿透)→ 子公司/关联公司
├── 分支机构 → 各地分公司/办事处
└── 上下游关联 → 供应商/客户
```

**企查查CLI调用链**:
```bash
# 股东/受益所有人(穿透到实控人)
qcc company get_beneficial_owners "企业名称"

# 分支机构(获取分公司列表)
qcc company get_branches "企业名称"

# 实际控制人
qcc company get_actual_controller "企业名称"

# 对外投资(子公司)
qcc company get_external_investments "企业名称"

# 年报(营收参考)
qcc company get_annual_reports "企业名称"
```

---

### 🔍 探针B:企业规模探测(按社保人数)

**目标**: 对"待开发"企业探测规模,锁定高价值客户

**核心逻辑**:
```
企业社保缴纳人数 → 企业规模等级
  1000人+  →  超大型(A级增量客户)
  300-1000人 →  大型(B级增量客户)
  50-300人  →  中型(C级增量客户)
  10-50人   →  小型(D级观察)
  <10人     →  微型(排除)
```

**探测流程**:
1. 百度地图Place语义搜索 → 获取企业名称+地址
2. 企查查CLI精确查询 → 获取参保人数+注册资本
3. **比对存量客户名单** → 标记绑定状态

---

### 🔍 探针A:竞品酒店探测

**目标**: 摸清酒店周边竞品格局,了解竞争态势

**威胁等级定义**:
- 🔴 强竞争:同档次/同商圈,评分≥4.6,热度>30
- 🟡 中竞争:档次相近,评分4.0~4.6,热度15~30
- 🟢 低竞争:档次不同或距离>15km

## 存量盘活策略(抄近道)【核心优先】

**绑定状态标记**:
- ✅ **德胧绑定**:已是德胧协议客户 → 重点做增购/续约
- ⚠️ **竞品绑定**:已是竞品协议客户 → 分析竞品策略,找差异化切入点
- 🔴 **待开发**:链上无协议,但与存量客户有关联 → **最高优先级增量**
- 🆕 **全新线索**:与存量客户无关联 → 放入队列,按规模/行业筛选

**存量盘活加分规则**:
| 场景 | 加分 | 说明 |
|------|------|------|
| 目标企业在存量客户上游供应链 | +30 | 已有商务关系,易切入 |
| 目标企业在存量客户下游 | +20 | 已有信任链,易推荐 |
| 目标企业与存量客户同一母公司 | +50 | 兄弟公司,决策链最短! |
| 存量客户最近1年有扩张/搬迁 | +20 | 换酒店窗口期 |
| 存量客户年度消费下滑 | +15 | 预警客户,需挽回 |
| 存量客户关键决策人变动 | +25 | 新人新机会 |

**⚠️ 重要提醒**:
- 探针探测到的企业规模再大,如果与存量客户无关联,都要降低优先级
- **有关系链的B级客户 > 无关联的A级客户**

## 五刷功能实现路径

### 五刷微信(暂时局限)
**现状**: 销售经理微信好友无法直接导出给AI
**过渡方案**:
1. 销售经理定期整理"高价值客户"名单(表格)→ 导入探针系统
2. 销售经理使用"备注+标签"功能 → AI辅助分析标签
3. 企微SCRM(未来方案)→ 数据可导出

### 企微SCRM方案(推荐)
**企业微信**可以:
- 成员好友数据导出(需管理员权限)
- 客户标签/备注可查询
- 客户群数据可分析
**接入方式**:
```
企业微信API → 客户数据 → 探针分析
```
**推荐动作**:
1. 德胧IT评估企微SCRM的可行性
2. 如可接入 → 探针系统直接对接企微客户池
3. 销售录入的客户信息 → AI辅助打标签
