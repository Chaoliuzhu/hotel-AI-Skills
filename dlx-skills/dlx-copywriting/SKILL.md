### 核心调性

| 维度 | 规范 |
|------|------|
| **视觉调性** | 温暖 + 精致。清晨阳光、海河波光、暖木色床品、滨海夜景 |

| **品牌定位** | 商务+度假双定位 |
| **地域特色** | 滨海新区、海河景观、鲤鱼门美食街 |

- **开元曼居系列**：中端商务

## 禁忌词（必须规避）

| 类别 | 禁用词 |
|------|--------|
| 绝对化 | 全国第一、绝对、极致、顶级、最好、完美 |
| 浮夸 | 奢华、奢侈、皇家、贵族、帝王 |
| 虚假 | 限时优惠（全网最低、史上最低）|
| 不符 | 夜店、派对、蹦迪、酒吧 |
| 未确认数据 | 具体房价、折扣比例未确认时禁止写 |

## 执行流程

### Step 1：确认需求

```
- 酒店/品牌：[哪个酒店]
- 平台：[小红书/朋友圈/抖音/大众点评]
- 受众：[商旅客人/亲子家庭/情侣/本地散客]
- 核心卖点：[产品/服务/体验]
- 目标：[种草/转化/品牌曝光]
```

### Step 2：品牌合规检查

- 调性是否符合"温暖精致"？
- 是否有禁忌词？
- 标签是否正确？
- 数据是否有依据？

### Step 3：文案撰写

**小红书体**（约300字）：
- 开篇有画面感（场景切入）
- 中段自然植入酒店产品/服务
- 结尾设置互动钩子
- 3-5个 Hashtag + 1-2个地点标签

**朋友圈体**（约100字）：
- 轻松自然，如同朋友分享
- 可不加 Hashtag
- 突出真实体验感

**抖音文案**（50字以内）：
- 前3秒有"钩子"
- 结尾有明确行动号召（CTA）

### Step 4：humanizer-zh 润色

写完后调用 humanizer-zh 去除 AI 味，让文案更自然。

## 容错机制

### 异常处理

- **humanizer-zh 调用失败（超时/不可用）**：跳过润色，直接输出原始文案；末尾标注「⚠️ 未经过humanizer-zh润色，建议手动处理AI味」
- **image_generate 生成失败**：输出文案时不包含配图prompt，改为文字描述风格；末尾标注「[配图建议：温暖手绘插画风，场景：xxx]」
- **用户信息不足（缺平台/受众/卖点）**：使用默认值填充——平台默认小红书、受众默认亲子家庭、卖点默认"海河景观+温暖服务"；向用户确认「已使用默认项，是否需要调整？」
- **品牌禁忌词检查发现问题**：自动标记并替换为合规替义词；保留原词便于用户查看替换逻辑
- **feishu-doc-enhancer 不可用**：降级为纯Markdown输出；告知用户「已生成Markdown版，可手动复制到飞书文档」

### 决策边界

| 情况 | 决策 |
|------|------|
| 平台不明确 + 用户未指定 | 默认输出小红书体，询问是否需要多平台版本 |
| 受众不明确 | 默认亲子家庭方向，兼顾商务元素 |
| 卖点不明确 | 用"海河景观+温暖服务+精致早餐"三大核心卖点填充 |
| 多平台需求 | 拆分输出，每平台独立一块，避免混杂 |
| 需求完全不明确 | 输出"需求确认模板"，引导用户逐步填写关键信息 |

### 检查点机制

每Step完成时输出状态快照：`✅ [Step N] 完成 | 平台：小红书 | 受众：亲子家庭 | 卖点：海河景观`
