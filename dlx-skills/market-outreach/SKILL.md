# SKILL.md — 德胧市场外拓选客技能

## 基本信息

- **技能名称**: market-outreach
- **中文名**: 市场外拓选客
- **版本**: v1.0
- **定位**: 酒店周边高净值客户开发工具。输入酒店地址或经纬度，输出周边企业优先级清单（A/B/C三级分层）+ 定向开发话术 + 对接宣推/选客SKILL的全流程工作流。
- **适用酒店**: 德胧旗下所有酒店（尤其是天津瑞湾开元名都等商务型酒店）
- **维护者**: 德胧AI龙虾军团

---


## 完整工作流（四步）

### Step 1：周边POI采集

**目标**: 通过地理API采集酒店周边目标企业名单。

**输入**: 酒店地址或经纬度 + 搜索半径

**工具**: 百度地图Place Search API（主）

**搜索关键词组合**:

| 类别 | 关键词 | 说明 |
|------|--------|------|
| 必选（高净值标签） | 世界500强、纳税大户、上市公司、央企、国企 | 直接匹配高价值客户 |
| 行业 | 写字楼、产业园区、商业综合体、产业园、科技园、孵化器 | 企业聚集地 |
| 补充行业 | 银行、证券、基金、投资、担保 | 金融类企业 |
| 补充行业 | 总部、集团、办事处 | 决策链短的大客户 |

**过滤规则**:
- ✅ 保留：写字楼、产业园区、商业综合体、上市公司、央企、国企、纳税大户
- ❌ 排除：住宅小区、学校、医院、幼儿园、政府机构（非经营类）、非营利组织
- ❌ 排除：纯住宅用途的"大厦"（需结合名称判断）

**输出**: 企业名称列表 + 地址 + 大致坐标

**容错**:
- 百度地图API超时（>5秒无响应）→ 自动切换高德地图POI API
- 百度地图Key无效（返回权限错误）→ 自动切换高德地图POI API
- 高德也失败 → 返回"数据采集中断"，提示用户手动补充或稍后重试

---

### Step 2：企业工商补全

**目标**: 获取Step1企业的工商信息，补全营收、员工规模、背景等字段。

**输入**: Step1输出的企业名称列表（最多50家/次）

**主数据源**: 阿里88查（免费额度，NLP语义查询）

**备1**: 爱企查API（需配置API Key）

**备2**: 风鸟企业（注册后5年内免费查询）

**备3**: 国家企业信用信息公示系统（免费但慢，需反爬处理）

**补全字段**:

| 字段 | 说明 | 来源优先级 |
|------|------|-----------|
| 年营收 | 万元/年 | 88查 → 爱企查 → 估算 |
| 员工规模 | 人 | 88查 → 爱企查 → 估算 |
| 行业分类 | 国民经济行业分类 | 爱企查 → 88查 |
| 纳税额 | 万元/年 | 88查（纳税大户专项）→ 爱企查 |
| 企业背景 | 央企/国企/民营/外资/上市公司 | 88查 → 爱企查 |
| 主营业务 | 文字描述 | 88查NLP → 爱企查 |

**过滤规则**:
- ❌ 排除：政府机关、事业单位、非营利组织（NPO）
- ❌ 排除：无法确认主营业务为非消费类（如纯物流仓储、纯农业种植）
- ✅ 保留：与酒店消费相关的企业（商务出差、会议培训、商务接待、差旅住宿需求）

**容错**:
- 88查无可用额度 → 自动切换爱企查API
- 爱企查Key无效 → 自动切换风鸟
- 全部外部API失败 → 使用本地缓存数据库（`memory/local-company-cache.md`，需手动更新）
- 企业名称模糊/重名 → 使用模糊匹配，标注置信度（高/中/低）
- 无营收数据 → 使用行业平均值估算（参考国家统计局行业平均营收）

---

### Step 3：CLV分层

**目标**: 对Step2补全的企业进行客户价值分层，对标选客SKILL的CLV体系。

**分层标准**:

| 等级 | 年预估CLV | 判断条件（满足任一） | 开发优先级 |
|------|-----------|---------------------|-----------|
| **A级** | ¥50,000+/年 | 世界500强、央企、上市公司年营收>10亿、纳税大户Top区域、持牌金融机构（银行/证券/保险） | 🔴 最高 |
| **B级** | ¥15,000~50,000/年 | 规上企业（年营收>2000万）、知名成长型公司（员工>200人）、高成长行业（医药/科技/金融科技） | 🟡 中 |
| **C级** | ¥5,000~15,000/年 | 普通企业、中小规模、年营收500万~2000万、会议/培训/团建需求可预估 | 🟢 一般 |

**CLV估算逻辑**:
```
年CLV = (年均出差频次 × 每次住宿天数 × 均价) + (年均会议场次 × 人均消费 × 参会人数)
A级企业：出差频次≥20次/年，每次≥2晚，均价≥400元
B级企业：出差频次8~20次/年，每次≥1晚，均价≥300元
C级企业：出差频次3~8次/年，每次1晚，均价≥200元
```

**特殊加分**:
- +总部型企业（决策链短）→ 升级
- +已有酒店协议客户的他店 → 降级（可能被竞品绑定）
- +近1年内有扩张/搬迁移交 → 升级（换酒店窗口期）

## 数据源主备切换规则（赛马机制）

### 地理数据源

```
优先级链：百度地图POI → 高德地图POI

触发降级条件：
1. 百度地图API返回error或超时（>5秒）
2. 百度地图Key无效（权限校验失败）
3. 百度返回结果为空（无POI数据）

降级操作：
→ 自动切换高德地图Place Search API，使用相同的关键词组合
→ 在输出中注明："地理数据来源：高德地图（百度地图降级）"
```

### 工商数据源

```
优先级链：阿里88查 → 爱企查API → 风鸟企业 → 国家信用公示系统

触发降级条件：
1. 88查无可用额度（返回quota_exceeded）
2. 88查返回错误（返回error或超时）
3. 爱企查API Key无效（权限错误）
4. 风鸟查询失败（反爬或服务不可用）

降级操作：
→ 逐级切换，直到有数据返回
→ 在输出中注明："工商数据来源：XXX（第X备选）"
→ 如果全部失败，启用本地缓存数据库
```

### 降级日志格式

每次数据获取时，在内部日志中记录：
```
[DATA_SOURCE] timestamp | source=<数据源> | status=<success/fallback> | fallback_reason=<原因>
```

## 与现有SKILL的集成

### 集成总览

```
market-outreach SKILL
├── 输出开发清单
│   ├── → dlx-customer-research（四类客群补充"周边企业客群"类型）
│   ├── → dlx-competitive-analysis（了解周边竞品酒店，防止被挖客户）
│   ├── → 选客SKILL（将清单作为"增量外部客户"输入，进行CLV五层复核）
│   └── → 宣推SKILL（按A/B/C级别生成定向投放内容方案）
```

### 1. 对接 dlx-customer-research

**集成点**: Step3 CLV分层后，输出中增加一行"客群类型归类"

**说明**: dlx-customer-research原本覆盖商旅客群、度假客群、本地散客、企业客户。本SKILL补充第5类——**周边企业客群**（从酒店周边3km内企业池中筛选出的高价值企业）。

**调用方式**: 在输出开发清单时，标注"客群类型：周边企业客群"，作为dlx-customer-research的增量输入。

### 2. 对接 dlx-competitive-analysis

**集成点**: Step1 POI采集时，增加一个"竞品酒店"关键词

**说明**: 在采集周边POI时，同步搜索"周边酒店"（使用"酒店"关键词），识别竞品酒店位置和档次，了解竞争格局，防止A/B级客户被竞品挖走。

**输入到dlx-competitive-analysis**: 竞品酒店名单 + 位置 + 档次定位

### 3. 对接选客SKILL（CLV五层复核）

**集成点**: Step4输出后

**调用方式**: 将本SKILL输出的A级/B级客户清单（企业名+预估年CLV+行业）作为选客SKILL的"外部增量客户"输入。

**选客SKILL操作**: 对这些企业进行CLV五层复核（原本的存量客户五层 + 外部新增客户五层），输出统一的客户分层视图。

### 4. 对接宣推SKILL

**集成点**: Step4输出后，根据A/B/C分级自动触发

**调用方式**:

| 客户分级 | 宣推内容方向 | 推送渠道 |
|----------|------------|---------|
| A级 | 品牌定制内容、高管圈层活动邀请、专属协议价值阐述 | 行政总监会晤、定制礼品寄送 |
| B级 | 团建产品介绍、会议场地推介、会员权益说明 | 企业微信公众号、HR社群 |
| C级 | 标准协议价产品、促销信息、会员注册引导 | 官网/小程序、短信/邮件 |

**宣推SKILL输出**: 每级客户生成3~5条定向内容方案，附带投放渠道建议。

## 容错机制汇总

| 故障场景 | 自动处理方式 | 降级路径 |
|---------|------------|---------|
| 百度地图POI超时/无效 | 自动切换高德地图 | 高德也失败 → 返回"采集中断"，提示手动补充 |
| 88查无可用额度 | 自动切换爱企查API | 爱企查失败 → 切换风鸟 → 失败 → 国家信用公示 |
| 全部工商API失败 | 启用本地缓存数据库 | `memory/local-company-cache.md`（手动更新） |
| 企业名称模糊/重名 | 模糊匹配，标注置信度 | 低置信度结果标注"⚠️ 需核实" |
| 无营收数据 | 使用行业平均值估算 | 在CLV列标注"[估算]" |
| 搜索半径超出10km | 提示超出上限，自动截断至10km | N/A |
| POI结果为空 | 提示"该区域数据较少"，建议扩大半径或调整关键词 | 扩大至5km重试 |

## 输出限制与注意事项

1. **数据时效性**: 工商数据通常有1~3个月延迟，重要决策前请核实最新数据
2. **搜索半径建议**: 3km适合城市商务区，郊区或度假型酒店可扩大至5~10km
3. **API调用频率**: 批量查询时注意遵守各平台QPS限制，避免被限流
4. **隐私合规**: 获取的企业信息仅用于酒店协议客户开发，不得用于其他商业用途
5. **结果复核**: A级客户的工商信息建议通过爱企查或88查进行二次核实
