## 核心流程（五步）

```
热点抓取 → 选题过滤 → 文案生成 → AI生图 → 输出发布草稿
```

### Step 1：热点抓取

使用 `web_search` 工具搜索：
- 关键词模板：`"天津 滨海新区 {日期} 热点 游玩"`
- 时间范围：最近3天内
- 重点关注：节假日活动、旅游攻略、美食推荐、本地新闻

### Step 2：选题过滤

六个核心维度判断价值：
1. **节假/节点**：周末短途游、情人节、五一国庆
2. **季节/时令**：春季赏花、夏季亲海、秋冬温泉
3. **本地热点**：鲤鱼门海鲜节、海河灯会、东疆亲海公园
4. **酒店产品**：客房升级、下午茶、婚宴会议套餐
5. **亲子/情侣**：家庭客群、情侣客群周末活动
6. **美食内容**：早餐、海鲜自助、商务套餐

**过滤原则**：与酒店**商务+度假**双定位不符的话题自动跳过（如夜店、酒吧类）

### Step 3：文案生成

按平台特性生成：

**小红书体**（约300字）：
- 开篇有画面感
- 中段自然植入酒店产品
- 结尾设置互动钩子
- 包含 3-5个 Hashtag + 1-2个地点标签

**朋友圈体**（约100字）：
- 轻松自然，如同朋友分享
- 可不加 Hashtag

**抖音/视频号文案**（50字以内）：
- 前3秒有"钩子"
- 结尾有明确行动号召

### Step 4：AI生图

调用 `image_generate` 批量生成配图：
- **小红书**：3:4 竖图，温暖明亮手绘插画风格
- **朋友圈**：16:9 横图，有氛围感的手绘插画
- **抖音封面**：16:9 横图，突出热点关键词的文字标题型
- 每个选题生成 2-3张备选

### Step 5：输出发布草稿

结构化内容包：
- 平台建议 + 最佳发布时间
- 各平台文案全文
- 配图（生成后上传到飞书云空间）
- Hashtag 使用建议

## 品牌调性指南

### 酒店基本信息

| 项目 | 内容 |
|------|------|
| 全称 | 天津瑞湾开元名都大酒店 |
| 位置 | 天津市滨海新区塘沽区，近海河入海口 |
| 定位 | 商务 + 度假 双定位 |
| 周边资源 | 海河景观带、滨海鲤鱼门美食街、东疆亲海公园、天津方特欢乐世界 |

### 视觉调性

- **温暖 + 精致**
- 画面应传递：清晨阳光洒进落地窗、海河波光与城市暮色交织、暖木色床品与精致早餐托盘的舒适感、滨海夜景的宁静氛围

### 文字调性

- 温暖且有画面感
- 避免生硬推销
- 具象场景引发共鸣
- 参考句式：
  - "把春天住进窗外"
  - "恰到好处的款待"
  - "看海的房间，不止看见海"

### 标签体系

**常驻必选标签**：
`#天津瑞湾开元名都` `#滨海新区酒店` `#天津酒店` `#开元名都`

**季节时令标签**（按需）：
`#天津春天` `#滨海春游` `#周末去哪儿` `#五一去哪`

**场景垂直标签**（按内容）：
`#酒店早餐` `#下午茶时光` `#海河景观` `#亲子酒店` `#婚宴场地`

### 禁忌词

| 类别 | 禁止使用 |
|------|----------|
| 绝对化用语 | 全国第一、绝对、极致、顶级、最好、完美 |
| 浮夸词汇 | 奢华、奢侈、皇家、贵族、帝王 |
| 虚假承诺 | 限时优惠（未确认时）、全网最低、史上最低 |
| 品牌不符元素 | 夜店、派对、蹦迪、酒吧类描述或视觉元素 |
| 未确认数据 | 未经确认的具体房价、折扣比例、套餐价格数字 |

### 配图风格规范

- **首选风格**：手绘插画风，温暖、明亮、有故事感和画面感
- **推荐色调**：暖米色、浅金色、淡蓝色等柔和明亮色系
- **禁止风格**：写实摄影风、明显低质量合成感的图片

## 容错机制

### 异常处理

- **web_search 热点搜索失败（无结果或网络错误）**：使用内置的时令热点日历作为内容来源；标注「⚠️ 实时热点未获取，使用时令热点替代，今日建议：[对应主题]」
- **image_generate 生成配图失败**：输出文案时不包含配图，提供文字版配图prompt；标注「[配图建议：温暖手绘插画风，场景：xxx]」
- **feishu-doc-enhancer 调用失败**：降级为纯Markdown格式输出内容包；标注「⚠️ 精美文档生成失败，已输出Markdown版」
- **热点与品牌调性不符（夜店/酒吧类话题）**：自动过滤，标注「已跳过：xxx（与商务+度假定位不符）」；自动补入下一个合适的时令热点
- **用户未指定日期（默认"今日"但时间已过）**：自动顺延至下一个可发布时间（明天08:30）；或询问用户「今日内容是否需要顺延至明天发布？」

### 决策边界

| 情况 | 决策 |
|------|------|
| 实时热点获取失败 | 使用时令日历保证产出，内容方向标注"时令热点" |
| 用户未指定平台 | 默认输出小红书+朋友圈双平台，抖音可选 |
| 热点窗口期已过 | 自动跳过，顺延至下一个时令节点 |
| 配图生成失败 | 提供文字prompt替代，不阻塞文案输出 |
| 内容包过于冗长 | 按平台拆分输出，每平台独立消息 |

### 检查点机制

每完成一个平台的内容生成输出状态快照：`📍 平台[X/3] 完成 | 小红书：✓文案+✓配图prompt | 朋友圈：✓文案 | 抖音：✓文案`
